# TH99 Pro keymap restore utility

A separate recovery tool (`th99_keymap_restore.py`) that re-applies saved
keymaps over the `MI_02` config channel — for example after a factory reset. It
is **not** part of the usage-display pipeline.

The keymap payload begins at byte 8 of the 64-byte packet, after the header's
final flag and reserved byte.

No command is sent in the normal/default mode:

```powershell
python .\th99_keymap_restore.py
```

The dry run validates both captures, confirms the final `AA 22` and `AA 26`
transactions can be regenerated byte-for-byte, compares them to the captured
`AA 12` and `AA 16` read-back, and shows the intended mapping differences.

Create a corrected, offline JSON target without touching the keyboard:

```powershell
python .\th99_keymap_restore.py --export-target .\th99-keymap-target.json
```

Live mode is intentionally not documented as a casual copy/paste command. It
requires `--execute-restore`, an exact acknowledgement token shown by
`--help`, the reviewed capture hashes, exactly one TH99 Pro `MI_02` interface,
and successful full-table read-back verification. Obtain explicit approval
immediately before using it.

Live-mode command whitelist:

- Read: `AA 12`, `AA 16`
- Write: `AA 22`, `AA 26`

The utility contains no factory-reset (`AA 0F`), TFT (`AA 50`), firmware,
macro, or lighting operation.
